## 🚀 使用方式

### 在桌面应用中使用搜索

#### 打开搜索框
- **快捷键**：`Cmd+K` (Mac) 或 `Ctrl+K` (Windows/Linux)
- **点击按钮**：点击左侧栏"搜索笔记..."按钮

## 🔧 技术架构

### 前端流程

```
用户输入
    ↓
300ms 防抖
    ↓
调用 search_notes (Tauri)
    ↓
后端返回搜索结果
    ↓
前端格式化显示
    ↓
用户选择笔记
    ↓
更新全局状态 (selectNote)
    ↓
关闭搜索框
    ↓
自动打开选中笔记
```

### 后端流程

```
接收 search_notes 请求
    ↓
提取查询字符串
    ↓
Tantivy 全文搜索
    ↓
中文分词处理
    ↓
返回前 50 条结果
    ↓
BM25 相关性排序
    ↓
返回给前端
```

## 📋 API 文档

### 搜索命令

#### `search_notes`
```typescript
invoke('search_notes', { query: string })
  → Promise<SearchResult[]>

// 参数
query: 搜索关键词

// 返回
{
  note_id: string;
  title: string;
  snippet: string;
  score: number;      // 0.0 ~ 1.0
  updated_at: number; // 毫秒时间戳
}[]
```

#### `search_notes_fuzzy`
```typescript
invoke('search_notes_fuzzy', { query: string })
  → Promise<SearchResult[]>

// 支持模糊匹配和容错搜索
```

#### `search_in_note`
```typescript
invoke('search_in_note', { 
  noteId: string, 
  query: string 
})
  → Promise<SearchResult[]>

// 在特定笔记中搜索
```

#### `search_notes_advanced`
```typescript
invoke('search_notes_advanced', { 
  query: string,
  limit: number,    // 结果数量限制
  offset: number    // 分页偏移
})
  → Promise<SearchResult[]>

// 支持分页的高级搜索
```

## 🐛 故障排查

### 搜索无结果

**可能原因**：
1. 笔记未被索引
2. 搜索词过于特殊
3. 后端搜索服务未启动

**解决方案**：
- 确认后端服务正常运行
- 检查笔记是否已创建
- 尝试更换搜索词
- 查看浏览器控制台错误日志

### 搜索很慢

**可能原因**：
1. 笔记数量过多
2. 系统资源不足
3. 搜索词过长

**解决方案**：
- 减少搜索结果限制
- 添加更多特定的搜索词
- 检查系统 CPU/内存使用情况

### 快捷键冲突

**如果 Cmd+K 不工作**：
- 检查系统是否占用此快捷键
- 查看浏览器扩展是否冲突
- 尝试通过 UI 按钮打开搜索

## 📝 使用场景

### 场景 1：快速查找笔记

```
用户想找 "React Hook" 相关的笔记

1. 按 Cmd+K 打开搜索
2. 输入 "React"
3. 系统自动分词：[React, Hook]
4. 返回相关笔记列表
5. 点击打开目标笔记
```

### 场景 2：模糊搜索

```
用户记不清完整标题

1. 按 Cmd+K 打开搜索
2. 输入 "学"
3. 系统返回所有包含"学"的笔记
4. 精确选择目标笔记
```

### 场景 3：笔记内搜索

```
用户想在特定笔记中搜索内容

1. 打开笔记后按 Cmd+K
2. 输入搜索词
3. 系统会在该笔记中搜索
4. 快速定位内容
```
