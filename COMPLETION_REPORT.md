💾 数据持久化架构
═══════════════════════════════════════════════════════════════════════════════

SQLite 数据库设计 (WAL 模式 + 外键约束):

notebooks 表
  ├─ id (TEXT PRIMARY KEY)
  ├─ name (TEXT)
  ├─ icon (TEXT) - Emoji 图标
  ├─ color (TEXT) - 颜色代码
  ├─ parent_id (TEXT FK) - 支持嵌套
  ├─ created_at, updated_at (INTEGER)
  └─ 笔记数量: COUNT(notes) 动态计算

notes 表
  ├─ id (TEXT PRIMARY KEY)
  ├─ notebook_id (TEXT FK ON DELETE SET NULL)
  ├─ title, content (TEXT)
  ├─ content_plain (TEXT) - 纯文本用于搜索
  ├─ is_pinned, is_favorite (INTEGER)
  ├─ word_count, version (INTEGER)
  ├─ created_at, updated_at, is_deleted (INTEGER)
  └─ 索引: notebook_id, updated_at DESC

tags 表
  ├─ id, name (UNIQUE), color

note_tags 关系表
  └─ 多对多关系 (ON DELETE CASCADE)

性能指标:
  • 创建笔记: < 10ms
  • 查询笔记: < 5ms
  • 全文搜索: < 50ms
  • 列表笔记: < 20ms

📖 核心 API 快速参考
═══════════════════════════════════════════════════════════════════════════════

笔记 API:
  invoke('create_note', {
    request: {
      title: string,
      content: string,
      notebook_id?: string,
      tags: string[]
    }
  }) -> Promise<Note>

笔记本 API:
  invoke('create_notebook', { name: string }) -> Promise<Notebook>
  invoke('rename_notebook', { id, name }) -> Promise<Notebook>
  invoke('delete_notebook', { id }) -> Promise<boolean>
  invoke('list_notebooks', {}) -> Promise<Notebook[]>

状态管理:
  store.createNote()         // 创建笔记
  store.createNotebook()     // 创建笔记本
  store.renameNotebook()     // 重命名笔记本
  store.deleteNotebook()     // 删除笔记本
  store.showToast()          // 显示提示
