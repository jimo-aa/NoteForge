## 六、架构建议与技术债务

### 6.1 建议纳入 V1.0 的技术改进

| 改进项 | 原因 | 建议时机 |
|--------|------|:--------:|
| 共享 JWT Secret → 独立 Secret 每服务 | 安全隐患 | P0 |
| 密码学 Salt 持久化 | 加密跨会话不可用 | P0 |
| 后端统一错误码体系 | 当前各 Controller 错误格式不统一 | P1 |
| Rust 引擎错误类型标准化 | 当前大量 `String` 错误传递 | P1 |
| ESLint 警告清零（51 条） | 代码质量 | P1 |
| 前端类型定义去重 | `Note` vs `NoteResponseItem` 部分重叠 | P2 |
| 前端状态管理重构 | Context 模式在复杂场景性能下降 | P2 |
| 测试基础设施统一 | Rust/Vitest/JUnit 各自为政 | P2 |

### 6.2 明确不纳入 V1.0 的功能

| 功能 | 原因 | 建议版本 |
|------|------|:--------:|
| Flutter 移动端 | 额外维护成本，V1.0 聚焦桌面 | V2.0 |
| Next.js Web 端 | 同上 | V2.0 |
| AI 写作助手 | 需 LLM API 集成 + 合规 | V2.0 |
| AI 自动标签 | 需 ML 模型 | V2.0 |
| AI 知识图谱 | 与 GraphView 联动，需 NLP | V2.0 |
| 实时协作编辑 | CRDT 复杂度高 | V3.0 |
| 社区插件系统 | 需沙箱 + SDK | V3.0 |
