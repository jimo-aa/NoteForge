## 二、下一阶段建议方案

### 方案 A：多平台扩展（推荐）

```
目标：补齐 Web + 移动端，实现"一次编写，三端运行"
周期：8-10 周
```

| Sprint | 内容 | 工作量 |
|:------:|------|:------:|
| S1 | **Next.js 15 Web 端 MVP**：项目初始化、共享 API 客户端、笔记 CRUD、Markdown 编辑 | 3w |
| S2 | **Web 端完善**：搜索、标签、笔记本管理、AI 功能、PWA 离线支持 | 2w |
| S3 | **Flutter 移动端 MVP**：项目初始化、API 集成、笔记 CRUD、离线缓存 | 3w |
| S4 | **三端同步打磨**：同步一致性测试、UI 适配、发布准备 | 2w |

**价值：** 补齐产品核心承诺，覆盖桌面+Web+移动全平台
**风险：** Web 端可复用 React 经验，Flutter 需学习 Dart/新框架
**ROI：** 高 — 产品形态完整化，用户覆盖翻倍

### 方案 B：协作编辑（激进）

```
目标：实时协作编辑 + 分享权限系统
周期：8-10 周
```

| Sprint | 内容 | 工作量 |
|:------:|------|:------:|
| S1 | **CRDT 数据结构**：Rust core 集成 `automerge`/`yrs` CRDT、操作转换 | 3w |
| S2 | **WebSocket 实时同步**：协作后端服务、OT/CRDT 冲突解决 | 2w |
| S3 | **前端协作 UI**：光标位置同步、选中高亮、协作者状态指示 | 2w |
| S4 | **分享 + 权限**：邀请链接、角色(R/W)、权限管理 API | 2w |

**价值：** 差异化核心功能，Notion/Obsidian 级协作
**风险：** CRDT 实现复杂度高，WebSocket 基础设施需重新设计
**ROI：** 中高 — 协作是笔记应用关键差异化，但开发周期长

目标：打磨现有桌面端 + 完善测试 + 性能极致优化

## 三、推荐方案：A → B → C 分阶段执行

### 3.1 推荐理由

基于以下分析，建议采用 **方案 A 为主，并行处理技术债务** 的策略：

1. **产品愿景对齐**：README 明确承诺"全平台"，Web 和移动端是核心交付
2. **后端已就绪**：5 个微服务 + API Gateway 已完整搭建，Web/移动端可直接复用
3. **用户价值高**：Web 端可立即扩大用户覆盖范围
4. **技术复用度高**：React 组件库、状态管理、AI 服务全部可直接复用
5. **PWA 策略**：先用 Web PWA 覆盖移动端，Flutter 移动端可以延后

### 3.2 V3.0 详细计划

```
Phase: V3.0 — 全平台扩展
周期: 8-10 周
优先策略: Web优先 → PWA移动覆盖 → 协作编辑
```

#### Milestone 1: Web 端 MVP（3 周）

| 周次 | 任务 | 交付物 |
|:----:|------|--------|
| W1 | Next.js 15 项目初始化 + 共享 API 层 + Auth 页面 | `web/` 项目骨架、登录/注册 |
| W2 | 笔记列表 + Markdown 编辑器(复用 TipTap) + 笔记本管理 | 笔记 CRUD、目录树 |
| W3 | 搜索 + 标签 + 同步 + AI 工具栏 | 全文检索、AI 写作、基础同步 |

**复用资产：**
- `desktop/src/services/aiService.ts` → 共享 API 客户端层
- `desktop/src/components/Editor/RichTextEditor.tsx` → TipTap 编辑器直接复用

- `desktop/src/types/index.ts` → 类型定义共享
- `desktop/src/i18n/` → 国际化文件共享

#### Milestone 2: PWA + 技术债务（2 周）

#### Milestone 3: 实时协作（3 周）

## 四、架构变更

### 4.1 Web 端架构 (Next.js 15)

```
web/                          # Next.js 15 App Router
├── app/
│   ├── (auth)/               # 登录/注册 (复用 backend user-service API)
│   │   ├── login/
│   │   └── register/
│   ├── (main)/               # 主应用 (需认证)
│   │   ├── notes/            # 笔记列表 + 编辑器
│   │   ├── notebooks/        # 笔记本管理
│   │   ├── tags/             # 标签管理
│   │   └── search/           # 搜索页面
│   ├── api/                  # BFF 代理层 (可选)
│   └── layout.tsx            # 根布局
├── components/               # 共享组件 (可复用 desktop 组件)
│   ├── Editor/               # TipTap 编辑器 (直接复用)
│   ├── Sidebar/              # 侧边栏
│   └── Common/               # Toast/ContextMenu/ErrorBoundary
├── lib/                      # 工具函数 + API 客户端
│   ├── api-client.ts         # 后端 API 封装
│   └── types.ts              # 共享类型 (引用 desktop/src/types)
└── public/                   # 静态资源 + PWA manifest
```

### 4.2 组件复用策略

| 组件 | desktop 路径 | Web 复用方式 |
|------|-------------|-------------|
| RichTextEditor | `components/Editor/RichTextEditor.tsx` | 直接复制，依赖一致 |
| AIToolbar | `components/Editor/AIToolbar.tsx` | 直接复制 |
| SearchBox (逻辑) | `components/Sidebar/SearchBox.tsx` | 提取共享逻辑到 `shared/` |
| NoteList | `components/Sidebar/NoteList.tsx` | Web 重写(不用虚拟滚动) |
| types | `types/index.ts` | 提取到共享包或直接引用 |
| aiService | `services/aiService.ts` | 直接复制 |
| i18n locales | `i18n/locales/` | 直接复制 |
| styles | `styles/globals.css` | CSS 变量体系复用 |

## 六、决策建议

### 立即开始 (Sprint 1)

| 任务 | 理由 |
|------|------|
| **Next.js 项目初始化** | 基础设施，阻塞其他任务 |
| **共享 API 客户端层** | 所有平台都需要，先做最值 |
| **TipTap 编辑器组件化** | 桌面端已有，提取为独立包供 Web 复用 |

### 延后决策 (Sprint 3 前定)

| 决策 | 选项 | 建议 |
|------|------|:----:|
| CRDT 方案 | `yrs` (Rust) / automerge / Liveblocks | `yrs` — Rust 原生，与 core 集成最好 |
| 协作后端 | 独立服务 / 并入 note-service | 独立协作服务 (collab-service) |
| 移动端策略 | Flutter / React Native / PWA | 先用 PWA 覆盖，Flutter 留到 V4 |

## 七、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:----:|:----:|----------|
| Web 端与桌面端行为不一致 | 中 | 高 | 共享 API 客户端 + 组件库 + E2E 测试 |
| CRDT 合并冲突复杂 | 中 | 高 | 先用简单锁定机制，逐步升级到 CRDT |
| PWA 离线体验不如原生 | 低 | 中 | 先用 Service Worker 缓存 API 响应 |
| Flutter 学习曲线 | 高 | 中 | Web PWA 作为过渡方案 |
