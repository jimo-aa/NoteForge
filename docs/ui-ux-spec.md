## 交互流程

### 笔记生命周期

```
用户操作                    UI 反馈                    数据流
────────────────────────────────────────────────────────────
点击"新建笔记"             弹出 Modal → 选择模板        Rust Core
填写标题 → 点"创建"        笔记列表出现新条目            SQLite INSERT
                           编辑器打开新笔记
                           └ 自动滚动到侧边栏新条目

在编辑器中输入              实时显示 Markdown 预览       本地渲染
                           状态栏显示"未保存"
                           自动保存 (300ms debounce)    SQLite UPDATE

点击侧边栏另一笔记          编辑器切换笔记                SQLite SELECT
                           恢复滚动位置
                           保留未保存变更提示

拖动笔记到笔记本            ← 笔记移动 (Phase 2)         UPDATE notebook_id
```

## 交互细节

### 自动保存

```
用户打字 → 启动 300ms debounce
     ↓
 无新输入 → 触发保存
     ↓
 状态栏: "已保存" ✓  (绿色)
 状态栏: "保存中..." (灰色)
 状态栏: "未保存"   (红色，出错时)
```

### 搜索体验

```
输入文字 → 200ms debounce → 过滤侧边栏列表
  → 匹配标题 / 标签 / 内容摘要
  → 实时显示匹配数量
  → 无结果时显示空状态提示
```

### 暗色模式

```
侧边栏已经为深色 (#1e1e2e)
编辑器区域可切换暗色:
  ┌ 工具栏太阳/月亮图标
  └ 编辑器 bg → #1a1a2e, text → #cdd6f4
  
自动跟随系统: prefers-color-scheme
```

## 设计原则

1. **离线优先** — 所有操作无需网络，UI 即时响应
2. **渐进式** — 常用功能一步到达，高级功能逐步发现
3. **一致性** — 统一的间距、色彩、圆角系统
4. **信息密度** — 侧边栏高密度列表，编辑器专注写作
5. **键盘优先** — 常用操作都有快捷键
