```
┌─────────────────────────────────────────────────────────┐
│                  NoteForge AI 能力矩阵                     │
├──────────────┬──────────────┬──────────────┬──────────────┤
│  ✍️ 写作助手  │  🏷️ 智能标签  │  🔍 智能搜索  │  🧠 知识图谱  │
├──────────────┼──────────────┼──────────────┼──────────────┤
│ 自动补全     │ 自动分类     │ 语义搜索     │ 实体提取     │
│ 续写         │ 关键词提取   │ 混合搜索     │ 关系发现     │
│ 改写润色     │ 批量标注     │ 多轮问答     │ 可视化       │

│              │              │              │              │
```

## 二、LLM 路由策略

```
请求类型         → 模型选择             → 延迟    → 成本
───────────────────────────────────────────────────────
简单补全         → 本地 Phi-3-mini      ~500ms   ≈ ¥0
标签/关键词       → 本地 BGE-micro      ~200ms   ≈ ¥0
摘要/改写         → Claude 3 Haiku      ~2s      ≈ ¥0.003
周报/长篇写作     → Claude 3.5 Sonnet   ~5s      ≈ ¥0.01
复杂问答/RAG      → GPT-4o / DeepSeek   ~8s      ≈ ¥0.03
深度推理           → o3-mini / DeepSeek-R1 ~15s   ≈ ¥0.05
```

## 三、RAG 问答流程

```
用户提问
    │
    ▼
┌────────────────┐
│ 意图分类        │
│ · 事实性问题    │ → 直接 LLM 回答
│ · 基于笔记问题  │ → RAG 流程
│ · 创作性问题    │ → 写作助手
│ · 分析性问题    │ → 知识图谱
└────────────────┘
    │ (RAG 流程)
    ▼
┌────────────────┐
│ 1. 查询向量化   │ → text-embedding-3-small → 1536d
└───────┬────────┘
        ▼
┌────────────────┐
│ 2. 混合检索     │
│ ├── 语义: pgvector/Milvus 相似度 Top-20
│ └── 关键词: ES BM25 相关性 Top-20
│ → RRF 融合排序 → Top-5
└───────┬────────┘
        ▼
┌────────────────┐
│ 3. 上下文构建   │
│ 拼接 Top-5 笔记片段 + 用户问题
│ 控制 token 在 4000 以内
└───────┬────────┘
        ▼
┌────────────────┐
│ 4. LLM 生成答案 │
│ + 引用来源标注
│ + 相关笔记推荐
└────────────────┘
```
