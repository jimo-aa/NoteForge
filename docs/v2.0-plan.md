## 三、里程碑规划

### Milestone 1：基础设施加固（2 周）

```
目标：安全加固 + 技术债务清理 + 类型系统重构
包含：P0 #1, P1 #7-#8, TD-1/2/4
```

| 周次 | 任务 |
|:----:|------|
| W1 | JWT Secret 隔离（Vault/K8s Secret）；后端集成测试容器 |
| W2 | 前端状态管理重构 (Context → Zustand)；类型系统清理 |

**验收标准：**
- [ ] JWT Secret 每服务独立，可外部化配置
- [ ] 后端集成测试覆盖 Redis + PostgreSQL
- [ ] 前端状态管理按模块拆分，store 文件 ≤ 150 行
- [ ] 无重复类型定义

### Milestone 2：AI 核心能力（2 周）

```
目标：AI 写作 + 智能搜索 + 自动标签
包含：P1 #5-#6, P2 #9
```

| 周次 | 任务 |
|:----:|------|
| W3 | AI 写作助手 API 集成（OpenAI/Claude），Markdown 续写/改写 |
| W4 | 语义搜索（embedding + pgvector）；自动标签分类器 |

**验收标准：**
- [ ] 选中文本可触发 AI 改写/续写/翻译
- [ ] 语义搜索返回相关度排序结果
- [ ] 新建/编辑笔记自动推荐标签

### Milestone 3：多平台扩展（3 周）

```
目标：Web 端 MVP + 移动端 MVP
包含：P0 #3, P1 #4
```

| 周次 | 任务 |

| W6 | Flutter 移动端项目初始化 + 笔记 CRUD + 离线同步 |
| W7 | Web 端 PWA 支持 + 移动端 UI 打磨 |

**验收标准：**
- [ ] Web 端支持笔记 CRUD + 搜索 + 同步

## 四、架构变更

### 4.1 Web 端架构

```
web/                          # Next.js 15 App Router
├── app/                      # 路由页面
│   ├── (auth)/               # 登录/注册
│   ├── (main)/               # 主应用（笔记列表+编辑器）
│   └── api/                  # BFF 代理层
├── components/               # 共享组件
├── lib/                      # 工具函数 + API 客户端
└── stores/                   # 状态管理（与 desktop 共享类型）
```

### 4.2 移动端架构

```
mobile/                       # Flutter 3.x
├── lib/
│   ├── app/                  # 应用入口 + 路由
│   ├── core/                 # 网络/数据库/同步
│   ├── features/             # 按功能模块划分
│   │   ├── auth/
│   │   ├── notes/
│   │   ├── search/
│   │   └── sync/
│   └── shared/               # 共享组件
└── test/
```

### 4.3 AI 服务架构（新增）

```
backend/
└── ai-service/               # 新增 Spring Boot 服务
    ├── controller/           # AI 生成/搜索/标签端点
    ├── service/              # LLM 调用 + embedding + 分类
    ├── client/               # OpenAI / Claude 客户端
    └── config/               # 模型配置 + 速率限制
```

### 4.4 API Gateway（新增）

```
backend/
└── gateway/                  # Spring Cloud Gateway
    ├── routes/               # 路由配置
    ├── filter/               # 鉴权/限流/日志过滤器
    └── config/               # CORS/SSL/限流配置
```

## 六、关键决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|:----:|------|
| 桌面端状态管理 | Zustand / Jotai / Redux | → Sprint 1 评估 | Context 重构为轻量方案，待 Sprint 1 验证性能需求 |
| AI 模型选型 | OpenAI / Claude / 本地 LLM | → Sprint 2 定 | 取决于用户隐私需求，初期用云端 API |
| Web 端框架 | Next.js 15 / Remix | Next.js 15 | 团队已有经验，App Router 支持 RSC |
| 移动端框架 | Flutter / React Native | Flutter | 原定技术栈，Dart FFI 可重用 Rust core |
| API Gateway | Spring Cloud Gateway / Kong | Spring Cloud Gateway | 与现有 Spring 生态一致 |
| 搜索升级 | pgvector / Milvus / ES | pgvector (已有) + embedding | 减少基础设施复杂度 |

## 七、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:----:|:----:|----------|
| AI API 成本超支 | 中 | 高 | 设置用量上限 + 缓存常见请求 |
