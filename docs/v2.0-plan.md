## 八、Sprint 2 实施计划 — AI 核心能力

### 8.1 任务分解

| 步骤 | 任务 | 模块 | 预估 | 交付物 |
|:----:|------|:----:|:----:|--------|
| 1 | **AI 模型选型** | 架构 | 0.5d | 选择 LLM 提供商（OpenAI / Claude / 本地），确定 API 规范 |
| 2 | **创建 ai-service 模块** | 后端 | 1d | Spring Boot 子模块：controller/service/client/config |
| 3 | **AI 写作 API 端点** | 后端 | 1d | `POST /api/v1/ai/write` (续写/改写/翻译) + `POST /api/v1/ai/complete` (补全) |
| 4 | **桌面端 AI 面板 UI** | 桌面 | 2d | 编辑器内 AI 工具栏：选中文本后出现「续写」「改写」「翻译」按钮 |
| 5 | **Markdown 流式渲染** | 桌面 | 1d | SSE/WebSocket 流式接收 AI 回复，实时渲染到编辑器 |
| 6 | **语义搜索集成** | 后端 | 2d | embedding 端点 + pgvector 向量存储 + 混合搜索 (BM25 + 向量) |
| 7 | **语义搜索 UI** | 桌面 | 1d | SearchBox 增加语义搜索模式切换，向量结果排序展示 |
| 8 | **自动标签** | 后端+桌面 | 2d | LLM prompt 分类 → 标签建议 → 一键应用标签 |

### 8.2 AI 模型选型建议

| 模型 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| **OpenAI GPT-4o** | 成熟稳定，API 完善 | 按 token 计费，需海外网络 | 写作/改写（首选） |
| **Claude 3.5 Sonnet** | 长上下文，中文质量高 | 同上 | 翻译/总结 |
| **本地 LLM (llama.cpp)** | 零成本，数据不外传 | 硬件要求高(16GB+ VRAM) | 自动标签/分类（可离线） |

> **建议**：写作助手用 OpenAI/Claude（质量优先），自动标签用本地模型（成本优先）。

### 8.4 API 设计

```
POST /api/v1/ai/write
  Request:  { action: "continue"|"rewrite"|"translate", text: string, context?: string }
  Response: { content: string, model: string, tokens: number }  // SSE 流式

POST /api/v1/ai/complete
  Request:  { text: string, maxTokens?: number }
  Response: { content: string, model: string }  // SSE 流式

POST /api/v1/ai/tag
  Request:  { title: string, content: string }
  Response: { tags: string[] }

POST /api/v1/ai/embed
  Request:  { text: string }
  Response: { embedding: number[] }

GET /api/v1/ai/search?q=...&mode=semantic
  Response: { results: SearchResult[] }
```

### 8.5 验收标准

- [ ] 编辑器内选中文本，弹出 AI 工具栏
- [ ] 点击「续写」→ 流式生成 Markdown 内容插入光标处
- [ ] 点击「改写」→ 替换选中文本为 AI 改写版本
- [ ] 点击「翻译」→ 在选中文本下方插入中文/英文翻译
- [ ] 搜索框可切换语义搜索，结果按相关性排序
- [ ] 新建/编辑笔记时自动推荐 3-5 个标签
